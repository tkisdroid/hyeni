## 범위

이 문서는 `/goal`의 "앱 전체 기능, 메뉴, 탭, 버튼, 모달, 폼, 저장 동작, 백그라운드 작업, 권한 요청, Android 네이티브 기능, Supabase 테이블/함수/Realtime 구독" 점검 요구를 현재 코드베이스 기준으로 매핑한다.

## 기능별 위험 목록

1. 폴더블 접힘 상태는 현재 연결 기기 `device_state`가 DEFAULT만 지원해 실제 접힘 상태 검증이 불가능하다. 사용자가 2026-05-05 KST에 다음 검증 세션으로 이월했다.
2. 기기 전체 오프라인은 Samsung child가 airplane mode + Wi-Fi/data disable 뒤에도 `networkConnected=true`, `networkValidated=true`를 유지해 재현하지 못했다. 앱 단위 네트워크 차단 evidence는 확보했다.
3. Play Billing/Qonversion 실제 결제 플로우는 unit/web fallback 중심으로 검증되어 Android 결제 실검증이 별도 필요하다.
4. 프로필 사진/Storage는 main real Playwright에서 signed URL 정상/없는 object fallback은 통과했지만, 느린 네트워크 체감 검증은 수동 잔여다.

## 검증 계획

1. 코드/DB가 변경되면 `npm run test`, `npm run verify`, `npx playwright test --config=playwright.real.config.js`, `npm run build`, `npx cap sync android`, Android `assembleDebug`를 재실행한다.
2. Android 부모/자녀 2대로 normal smoke와 `--child-background-only` smoke를 재실행한다.
3. `node scripts/android-remote-listen-matrix.mjs --require-two --parent emulator-5554 --child R5CY40EE6QE`로 권한/기기 상태 evidence를 갱신한다.
4. 폴더블 접힘 검증은 실제 foldable 기기 또는 foldable AVD 준비 후 동일 matrix/smoke를 실행한다.
5. 프로필 사진 느린 네트워크 체감과 Play Billing/Qonversion 실제 결제 흐름은 별도 수동 실기기 세션에서 확인한다.
