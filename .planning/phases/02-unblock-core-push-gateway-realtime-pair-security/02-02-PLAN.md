**Supabase Realtime contract (STACK.md §Issue #2 + official docs):**
- `supabase.channel(name)` — name must be unique per connection; per-table naming `${table}-${familyId}` is stable enough
- `.on('postgres_changes', { event, schema, table, filter }, callback)` — ONE binding per channel recommended (D-B02)
- `.on('broadcast', { event }, callback)` — broadcast events stay on `family-{familyId}` channel (D-B06)
- `.subscribe((status, err) => ...)` — status values: `SUBSCRIBED`, `CHANNEL_ERROR`, `TIMED_OUT`, `CLOSED`
- Filters on non-PK columns (we filter on `family_id`) REQUIRE `REPLICA IDENTITY FULL` on source table, else Realtime silently drops UPDATE events (PITFALLS §Pitfall 2.2)
- `postgres_changes` payload shape: `{ eventType: 'INSERT'|'UPDATE'|'DELETE', new: {...}, old: {...} }`

    //
    // Phase 2 Stream B refactor (D-B02): per-table channel pattern to prevent
    // "one bad binding kills all bindings" failure mode (supabase-js#1917).
    //
    // Channel layout:
    //   family-{familyId}              — broadcast only (kkuk, location, remote_listen_*)  [D-B06]
    //   events-{familyId}              — postgres_changes: events        (RT-03)
    //   memos-{familyId}               — postgres_changes: memos         (RT-03)
    //   academies-{familyId}           — postgres_changes: academies
    //   saved_places-{familyId}        — postgres_changes: saved_places  (RT-01)
    //   family_subscription-{familyId} — postgres_changes: family_subscription  (RT-02)
    //   memo_replies-{familyId}        — postgres_changes: memo_replies  (RT-03)
    //
    // Each per-table channel has independent error handling — a broken binding
    // on one channel does NOT affect the others.

    function subscribeTableChanges(channelName, tableName, familyId, eventSpec, handler) {
      // eventSpec: "*" for all events, or "INSERT"/"UPDATE"/"DELETE"
      let retryCount = 0;
      const MAX_RETRIES = 10;
      const BASE_DELAY_MS = 2000;
      let retryTimer = null;
      let disposed = false;

      const ch = supabase
        .channel(channelName)
        .on("postgres_changes", {
          event: eventSpec,
          schema: "public",
          table: tableName,
          filter: `family_id=eq.${familyId}`,
        }, (payload) => {
          if (handler) handler(payload.eventType, payload.new, payload.old);
        })
        .subscribe((status, err) => {
          if (status === "SUBSCRIBED") {
            console.log(`[Realtime] Subscribed to ${channelName}`);
            retryCount = 0;
          }
          if ((status === "CHANNEL_ERROR" || status === "TIMED_OUT") && !disposed) {
            console.error(`[Realtime] ${channelName} ${status}`, err);
            if (retryCount < MAX_RETRIES) {
              const delay = Math.min(BASE_DELAY_MS * Math.pow(2, retryCount), 60000);
              retryCount++;
              console.log(`[Realtime] ${channelName} reconnecting in ${delay}ms (attempt ${retryCount}/${MAX_RETRIES})`);
              retryTimer = setTimeout(() => {
                if (disposed) return;
                try { ch.subscribe(); } catch { /* ignored */ }
              }, delay);
            } else {
              console.error(`[Realtime] ${channelName} max retries reached; disabled until page reload.`);
            }
          }
          if (status === "CLOSED") {
            console.log(`[Realtime] ${channelName} closed`);
          }
        });

      ch._dispose = () => {
        disposed = true;
        if (retryTimer) clearTimeout(retryTimer);
      };
      return ch;
    }

      const {
        onEventsChange,
        onAcademiesChange,
        onMemosChange,
        onMemoRepliesChange,
        onSavedPlacesChange,
        onFamilySubscriptionChange,   // NEW — RT-02 consumer may not exist yet; optional
        onLocationChange,
        onKkuk,
        onRemoteListenStart,
        onRemoteListenStop,
        onAudioChunk,
      } = callbacks;

      // ── Per-table postgres_changes channels (D-B02) ─────────────────────
      const eventsCh = subscribeTableChanges(
        `events-${familyId}`, "events", familyId, "*",
        onEventsChange);

      const academiesCh = subscribeTableChanges(
        `academies-${familyId}`, "academies", familyId, "*",
        onAcademiesChange);

      const memosCh = subscribeTableChanges(
        `memos-${familyId}`, "memos", familyId, "*",
        onMemosChange);

      const savedPlacesCh = subscribeTableChanges(
        `saved_places-${familyId}`, "saved_places", familyId, "*",
        onSavedPlacesChange);

      const familySubCh = subscribeTableChanges(
        `family_subscription-${familyId}`, "family_subscription", familyId, "*",
        onFamilySubscriptionChange);  // undefined handler is fine — events drop

      const memoRepliesCh = subscribeTableChanges(
        `memo_replies-${familyId}`, "memo_replies", familyId, "INSERT",
        onMemoRepliesChange
          ? (_eventType, newRow) => onMemoRepliesChange(newRow)
          : null);

      // ── Broadcast-only channel (D-B06) — kkuk, location, remote_listen_*
      const broadcastCh = supabase
        .channel(`family-${familyId}`)
        .on("broadcast", { event: "child_location" }, (payload) => {
          if (onLocationChange) onLocationChange(payload.payload);
        })
        .on("broadcast", { event: "kkuk" }, (payload) => {
          if (onKkuk) onKkuk(payload.payload);
        })
        .on("broadcast", { event: "remote_listen_start" }, (payload) => {
          if (onRemoteListenStart) onRemoteListenStart(payload.payload);
        })
        .on("broadcast", { event: "remote_listen_stop" }, (payload) => {
          if (onRemoteListenStop) onRemoteListenStop(payload.payload);
        })
        .on("broadcast", { event: "audio_chunk" }, (payload) => {
          if (onAudioChunk) onAudioChunk(payload.payload);
        })
        .subscribe((status) => {
          if (status === "SUBSCRIBED") console.log(`[Realtime] Subscribed to family-${familyId} (broadcast)`);
          if (status === "CLOSED") console.log(`[Realtime] family-${familyId} closed`);
        });
      broadcastCh._dispose = () => { /* no retry state on broadcast-only */ };

      // Composite handle — behaves like a single channel for the caller.
      const channels = [eventsCh, academiesCh, memosCh, savedPlacesCh, familySubCh, memoRepliesCh, broadcastCh];
      const composite = {
        _dispose: () => { channels.forEach(ch => { try { ch._dispose?.(); } catch { /* ignored */ } }); },
        _channels: channels,
      };
      return composite;
    }

    export function unsubscribe(compositeOrChannel) {
      if (!compositeOrChannel) return;
      // New composite handle from subscribeFamily
      if (Array.isArray(compositeOrChannel._channels)) {
        if (compositeOrChannel._dispose) compositeOrChannel._dispose();
        compositeOrChannel._channels.forEach(ch => {
          try { supabase.removeChannel(ch); } catch { /* ignored */ }
        });
        return;
      }
      // Legacy single-channel path (belt-and-suspenders in case callers still hold old channel refs)
      if (compositeOrChannel._dispose) compositeOrChannel._dispose();
      try { supabase.removeChannel(compositeOrChannel); } catch { /* ignored */ }
    }

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → Realtime WS | Authenticated session JWT (ES256) presented at channel join; Supabase Realtime validates via JWKS |
| Realtime → postgres logical replication slot | Service-internal trust; replication slot is in Supabase-managed subnet |
| postgres_changes payload → client | RLS re-evaluated per-row per-client at emission; row only dispatched if client's role has SELECT on that row |
| publication membership → authenticated roles | Public membership (anyone with WS access to `supabase_realtime` + relevant RLS SELECT) — not a sensitive property |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-09 | Information disclosure | saved_places.location (lat/lng) via Realtime | mitigate | Supabase Realtime emits postgres_changes ONLY to clients whose session JWT passes the table's RLS `SELECT` policy (`sp_select_family` at `20260418000006_saved_places.sql`). `REPLICA IDENTITY FULL` makes full row available for filter evaluation but does not broaden RLS scope. Family-scope confinement preserved. |
| T-02-10 | Information disclosure | family_subscription.tier leaked cross-family | mitigate | Same as T-02-09 — family_subscription has RLS in `20260418000005_subscription_rls.sql`; Realtime respects it at emit time. |
| T-02-11 | Denial of service | `TooManyChannels` error on high-familyCount clients | accept | Supabase hosted limit is plan-dependent (typically 500 channels/connection on Pro+). Our 7 channels per family × 1 family = 7 per user session — far below limit. Documented in STACK.md §Issue #2. |
| T-02-12 | Tampering | Stale PostgREST REST cache (saved_places 404 after table exists) | mitigate | `NOTIFY pgrst, 'reload schema'` issued as part of the publication migration (Step 3 of up migration). Refreshes PostgREST REST endpoint cache. Confirmed observable post-migration via `fetchSavedPlaces` returning 200 instead of 404. |
| T-02-13 | Repudiation | Which table emitted a given postgres_changes event | accept | Each per-table channel has a unique topic name (`{table}-{familyId}`); payload includes `table` in `payload.table`. Client-side logging already records channel name + eventType. Acceptable audit for this phase. |
| T-02-14 | Spoofing | Client sending `phx_join` for another family's channel name | mitigate | Supabase Realtime server-side validates session JWT's `sub` against RLS policy at subscribe time; INSERT/UPDATE payloads filtered server-side by family_id + RLS. Client-side spoofing of channel name alone does not grant access to another family's data. |
| T-02-15 | Denial of service | Broadcast channel flood (kkuk spam) | accept | Broadcast channel `family-{familyId}` is NOT rate-limited in Phase 2. KKUK-03 (Phase 5 Stream C) introduces server-side 5s/sender cooldown. Accept for Phase 2; Phase 5 closes. |
