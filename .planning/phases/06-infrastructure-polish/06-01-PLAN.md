Create `.github/workflows/android-apk.yml` with this exact shape:

```yaml
name: Android APK

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]
  workflow_dispatch:

concurrency:
  group: android-apk-${{ github.ref }}
  cancel-in-progress: true

permissions:
  contents: read

jobs:
  build:
    name: Build Android APK
    runs-on: ubuntu-latest
    timeout-minutes: 30

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup JDK 17 (Temurin)
        uses: actions/setup-java@v4
        with:
          distribution: temurin
          java-version: '17'

      - name: Setup Android SDK
        uses: android-actions/setup-android@v3

      - name: Setup Node.js 20
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Build web bundle (vite)
        run: npm run build

      - name: Capacitor sync (Android)
        run: npx cap sync android

      - name: Grant gradlew execute permission
        working-directory: android
        run: chmod +x ./gradlew

      # Release signing is opt-in. If ANDROID_KEYSTORE_BASE64 is present we
      # decode it into android/app/release.keystore and run assembleRelease.
      # Otherwise we run assembleDebug so CI stays green without secrets.
      - name: Decide build variant
        id: variant
        run: |
          if [ -n "${{ secrets.ANDROID_KEYSTORE_BASE64 }}" ]; then
            echo "task=:app:assembleRelease" >> "$GITHUB_OUTPUT"
            echo "apk_glob=android/app/build/outputs/apk/release/*.apk" >> "$GITHUB_OUTPUT"
            echo "signed=true" >> "$GITHUB_OUTPUT"
          else
            echo "task=:app:assembleDebug" >> "$GITHUB_OUTPUT"
            echo "apk_glob=android/app/build/outputs/apk/debug/*.apk" >> "$GITHUB_OUTPUT"
            echo "signed=false" >> "$GITHUB_OUTPUT"
          fi

      - name: Materialize release keystore (signed path only)
        if: steps.variant.outputs.signed == 'true'
        env:
          KEYSTORE_B64: ${{ secrets.ANDROID_KEYSTORE_BASE64 }}
        run: |
          echo "$KEYSTORE_B64" | base64 -d > android/app/release.keystore
          # Gradle looks for these via -P<name> below.

      - name: Gradle build
        working-directory: android
        env:
          ORG_GRADLE_PROJECT_HYENI_KEYSTORE: ${{ github.workspace }}/android/app/release.keystore
          ORG_GRADLE_PROJECT_HYENI_KEYSTORE_PASSWORD: ${{ secrets.ANDROID_KEYSTORE_PASSWORD }}
          ORG_GRADLE_PROJECT_HYENI_KEY_ALIAS: ${{ secrets.ANDROID_KEY_ALIAS }}
          ORG_GRADLE_PROJECT_HYENI_KEY_PASSWORD: ${{ secrets.ANDROID_KEY_PASSWORD }}
        run: ./gradlew --no-daemon ${{ steps.variant.outputs.task }}

      - name: Upload APK artifact
        uses: actions/upload-artifact@v4
        with:
          name: hyeni-calendar-apk-${{ github.sha }}
          path: ${{ steps.variant.outputs.apk_glob }}
          if-no-files-found: error
          retention-days: 14
```

Notes the executor should respect:
- The `Decide build variant` step is the reason this plan ships GREEN on day one without any GitHub Secrets. Phase 7 NATIVE-02 will add signing secrets to flip it to `assembleRelease` — no workflow edit required.
- Do NOT wire `apply signingConfigs` into `android/app/build.gradle` in this plan. Signing config belongs to NATIVE-02. If `assembleRelease` is selected but the gradle file has no signing block, that's expected — Phase 7 will add it. The debug path stays usable regardless.
- Pin all `uses:` actions to major version (`@v4`, `@v3`) per GitHub-recommended practice. Do not use `@latest` or untagged.
- `actions/setup-node@v4` with `cache: 'npm'` uses the package-lock.json that already exists in repo root.
- `timeout-minutes: 30` caps runaway gradle builds.
- `concurrency` block cancels an in-flight run on the same branch when a new commit lands.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub PR author -> workflow | Forked PRs cannot read secrets (GitHub default); `pull_request` event runs the debug path only. |
| GitHub Secrets -> runner env | Keystore + passwords flow through `ORG_GRADLE_PROJECT_*` env vars into gradle. |
| Runner -> Artifacts storage | Built APKs are stored unsigned (debug) or signed (release) for 14 days; accessible to repo collaborators. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01-01 | Tampering | `android-apk.yml` pinned actions | mitigate | Pin `actions/*` and `android-actions/setup-android` to major versions (`@v4`, `@v3`). Do not use unpinned refs. |
| T-06-01-02 | Information Disclosure | Keystore secrets in logs | mitigate | Pass secrets only via `env:` on the gradle step; never `echo` them. `base64 -d` decodes to a file path, secret string never appears in `set -x` output. |
| T-06-01-03 | Elevation of Privilege | Fork PR reading secrets | accept | GitHub's default policy already denies secrets to fork PRs. Workflow's `pull_request` trigger therefore follows the debug path on forks — no secret access attempted. |
| T-06-01-04 | Denial of Service | Runaway gradle build burning Actions minutes | mitigate | `timeout-minutes: 30` + `concurrency` cancel-in-progress. |
| T-06-01-05 | Repudiation | Unsigned APK distributed as if official | accept | Debug APKs are clearly labeled `app-debug.apk` in the artifact. Production distribution goes through Play Console internal track (NATIVE-02) with signed release AAB, not via Actions artifacts. |
| T-06-01-06 | Spoofing | Malicious action in workflow | mitigate | Use only first-party (`actions/*`) and the well-known `android-actions/setup-android`. No ad-hoc third-party actions. |
</threat_model>
