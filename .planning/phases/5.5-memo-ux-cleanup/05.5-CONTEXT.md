## Origin

사용자가 2026-04-22 production 에서 memo 전송 기능이 "커서가 보이는 채로 입력 중단된 것처럼" 작동한다고 보고. 동시에 X(Twitter)/Thread 스타일의 깔끔한 말풍선 UI 를 명시적으로 요구.

## Root cause (from debug session `kkuk-memo-send-noop`)

두 개의 공존하는 memo 쓰기 경로 중 **legacy `public.memos` textarea 경로가 수신측에서 ghost 렌더링** 을 유발:

1. `src/App.jsx:2088-2143` `MemoSection` 상단 `<textarea>` 는 `memoValue` prop (= `memos[dateKey]`) 에 바인딩
2. `onMemoSend`/`onMemoBlur` (`src/App.jsx:6754-6784`) 가 여전히 `upsertMemo(familyId, dateKey, text)` 로 legacy 테이블에 기록
3. 수신측 `onMemosChange` Realtime 핸들러 (`src/App.jsx:4664-4678`) 에 `if (newRow.date_key === dateKeyRef.current) return` 무조건 skip 규칙이 있어, 수신자가 같은 날짜를 보고 있으면 해당 memo state 가 갱신되지 않음
4. 따라서 수신자의 textarea 는 수신자 자신이 이전에 입력하다 지운 텍스트(또는 30초 polling 으로 늦게 반영된 상대 텍스트) 그대로 남아 "입력 중단된 상태" 로 보임
5. 말풍선 경로 (`memo_replies` → `sendMemo` → `onMemoRepliesChange`, App.jsx:4740-4747) 는 정상 작동하므로 본 버그는 legacy textarea 쪽에 국한됨

Full evidence: `.planning/debug/kkuk-memo-send-noop.md` (status=diagnosed).

## Goal

1. **Ghost UX 제거**: 수신측이 실시간 채팅처럼 상대 메시지를 즉시 확인 가능
2. **X/Thread 스타일 UI**: 내가 보낸 메시지와 상대가 보낸 메시지가 말풍선으로 명확히 구분되고, 시간순으로 깔끔하게 배치
3. **MEMO-CLEANUP-01 조기 완료**: 원래 v1.2 또는 v1.3 으로 계획돼 있던 legacy `public.memos` write-path 제거를 본 phase 에서 수행 (shadow-30-day 정책은 v1.0 종료 시 무효화됨 — 2026-05-21 대기 없이 진행)

## Requirements

- **MEMO-FIX-01** — legacy `upsertMemo` 호출 제거: `src/App.jsx:6754-6784` `onMemoSend` · `onMemoBlur` 에서 `upsertMemo(familyId, dateKey, text)` 호출 삭제. 상단 textarea 는 read-only 로 전환하거나 레이아웃에서 제거 (디자인 결정은 /gsd-ui-phase 에서).
- **MEMO-FIX-02** — 잔여 Realtime skip 로직 정리: `onMemosChange` (App.jsx:4664-4678) 의 date-key skip 조건을 (a) 완전히 제거하거나 (b) 자신이 쓴 row 만 skip 하도록 좁힘. 쓰기 경로가 사라지면 이 핸들러의 부하도 거의 없어짐.
- **MEMO-FIX-03** — X/Thread 스타일 bubble UI:
  - 현재 `src/App.jsx:2151-2176` 의 bubble 렌더링이 `flexDirection: isMe ? "row-reverse" : "row"` 로 방향만 구분됨 → 여기에 X/Thread 벤치마크 수준의 spacing·avatar·timestamp·read-status 추가
  - 연속 메시지 grouping, 시간 헤더, 날짜 구분선, 부드러운 애니메이션 (스크롤 도착 시), 적절한 maxWidth 로 가독성 확보
  - 모바일 퍼스트 (Capacitor WebView 대상), dark-mode 호환
  - UI-SPEC 는 `/gsd-ui-phase 5.5` 에서 생성
- **MEMO-FIX-04** — 회귀 테스트: 부모·아이 두 세션을 띄워 (1) 전송자 본인 화면 즉시 반영, (2) 수신자 화면 <3초 내 반영, (3) textarea 잔상 부재, (4) grouping/timestamp 규칙 정상 동작 확인. `tests/e2e/memo-bubbles.spec.js` 신규 작성.
- **MEMO-FIX-05** — codex review gate: 모든 변경 diff 가 `codex review` 를 통과해야 commit 가능 (2026-04-22 user feedback memory)

## Success Criteria
