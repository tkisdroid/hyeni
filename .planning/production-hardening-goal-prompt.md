# 혜니캘린더 프로덕션 안정화 /goal 프롬프트

아래 프롬프트를 새 Codex 세션의 첫 메시지로 붙여 넣는다. 목표는 앱을 "그럴듯하게" 고치는 것이 아니라, 실제 코드/DB/브라우저/Android 기기에서 검증된 프로덕션 안정화 상태까지 반복해서 끌어올리는 것이다.

```text
/goal 혜니캘린더 앱을 프로덕션 레벨로 안정화한다.

역할:
너는 혜니캘린더 프로젝트의 프로덕션 안정화 담당 시니어 엔지니어다. 현재 코드베이스, Supabase DB, Realtime, Capacitor Android, 브라우저 UI, 연결된 Android 기기를 기준으로 실제 동작을 끝까지 구현하고 검증한다.

최상위 목표:
혜니캘린더 앱의 모든 기능과 화면 품질을 프로덕션 레벨로 끌어올린다. 특히 앱 전체 기능 점검, 다자녀별 데이터/기능 격리, 실시간 위치 반영, 홈/오늘 메뉴 동기화, 캘린더에서 선택 아이의 하루 이동경로 요약 기능, 주위소리 듣기 권한과 실행 흐름, 아이 기기 상태별 주위소리 실패 원인 진단, 프로필/설정 저장 로딩 UX, 회원가입/프로필 테마 색상 기반 아이콘 표시, 프로필 사진 로딩 성능, DB와 앱 기능별 최적화를 완성한다. 분석에서 멈추지 말고 필요한 코드 적용과 수정까지 완료한다. 실패한 항목은 원인을 특정한 뒤 수정, 재검증, 재확인을 반복한다. 검증되지 않은 항목이 남아 있으면 완료라고 말하지 않는다.

절대 원칙:
- AGENTS.md와 .planning 문서를 먼저 읽고 따른다.
- 현재 프로젝트의 실제 파일, 실제 타입, 실제 DB 스키마, 실제 API 응답을 확인한 뒤 수정한다.
- 추측 구현, placeholder, pseudo code, 생략 코드 금지.
- 현재 프로젝트에 없는 라이브러리 임의 추가 금지.
- src/App.jsx 모놀리스 분해 금지. 필요한 최소 범위만 수정한다.
- 기존 기능과 사용자 데이터를 깨지 않는다.
- DB 변경은 Supabase branch, migration hygiene, down migration, BEGIN/COMMIT, RLS 안전성을 지킨다.
- 수정 후 브라우저, 테스트, Supabase, Android 연결 기기에서 실제 검증한다.
- UI 수정은 Playwright로 실제 화면 표시, 스크린샷, 주요 요소 상태를 확인한다.
- 검증 실패 시 완료 처리하지 말고 원인 분석 후 다시 수정한다.

1. 사전 조사와 앱 전체 기능 점검:
- AGENTS.md, .planning/PROJECT.md, .planning/ROADMAP.md, .planning/REQUIREMENTS.md, .planning/research/SUMMARY.md를 읽는다.
- package.json, Supabase 설정, migrations, src/App.jsx, src/lib/auth.js, src/lib/sync.js, src/lib/pushNotifications.js, Android 네이티브 파일을 확인한다.
- 부모/자녀 선택 흐름, selectedChild/child_id/family_id 상태 흐름, Supabase 테이블, Realtime 구독, 권한 요청, 저장 UI 상태를 실제 코드 기준으로 맵핑한다.
- 앱의 모든 기능, 메뉴, 탭, 버튼, 모달, 폼, 저장 동작, 백그라운드 작업, 권한 요청, Android 네이티브 기능, Supabase 테이블/함수/Realtime 구독을 기능 인벤토리로 작성한다.
- 회원가입, 로그인, 가족 생성, 아이 등록/선택, 페어링, 홈, 오늘, 캘린더, 일정 추가/수정/삭제, 위치, 하루 이동경로 요약, 주위소리 듣기, 프로필, 테마, 저장소/이미지, 알림/푸시, 결제/구독, 설정, 로그아웃/오류 상태를 빠짐없이 점검한다.
- 각 기능마다 "관련 파일", "관련 DB 테이블", "선택 아이 격리 여부", "권한/Realtime 여부", "로딩/실패 UI", "검증 방법"을 기록한다.
- 기능별 위험 목록과 검증 계획을 먼저 작성한다. 기능 인벤토리에 없는 기능은 완료 검증 대상에서 빠뜨리지 말고 추가한다.

2. 다자녀별 DB와 기능 격리:
- 첫 화면에서 아이를 선택한 뒤 앱 전체가 선택된 아이 1명에게만 상호작용하는지 확인한다.
- 일정, 홈, 오늘, 위치, 프로필, 테마, 알림, 주위소리 듣기, 저장/수정 기능이 다른 아이 데이터와 섞이지 않게 한다.
- 모든 DB 쿼리, mutation, 상태, 캐시, Realtime 구독, UI 표시가 selectedChild 또는 명시적 child_id 기준인지 확인한다.
- child_id/family_id 누락 쿼리와 전체 family 데이터가 잘못 노출되는 경로를 찾아 수정한다.
- 자녀 전환 시 이전 자녀의 위치, 로딩 상태, 입력값, 구독, 요청 상태가 남지 않도록 정리한다.
- RLS 정책과 DB 레벨에서도 다른 자녀 데이터 접근이 차단되는지 확인한다.

3. 위치 실시간 반영과 캘린더 하루 이동경로 요약:
- 아이 위치가 child_id 기준으로 DB에 저장/업데이트되는 흐름을 확인한다.
- 부모 앱에서 선택된 아이의 최신 위치가 실시간으로 반영되는지 확인한다.
- 홈, 오늘, 위치 관련 메뉴의 UI 요소가 같은 Realtime source 또는 안전한 동기화 경로를 통해 갱신되는지 확인한다.
- Realtime unsubscribe 누락, 중복 구독, stale state, child 전환 시 이전 위치 표시 문제를 수정한다.
- 위치 권한 없음, 위치 없음, 오래된 위치, 오프라인, 네트워크 실패, 빈 데이터 상태를 사용자에게 안전하게 표시한다.
- 캘린더에서 날짜를 선택하면 선택된 아이의 하루 이동경로 요약을 볼 수 있는 기능을 구현한다.
- 하루 이동경로 요약은 선택 아이와 선택 날짜 기준으로만 조회한다. 다른 아이 또는 다른 날짜 위치 데이터가 섞이면 안 된다.
- 실제 DB에 위치 히스토리 테이블/필드가 있는지 먼저 확인한다. 없거나 불충분하면 안전한 migration, RLS, index, realtime publication 필요 여부를 설계하고 구현한다.
- 이동경로 요약에는 가능한 범위에서 첫 위치/마지막 위치, 시간대별 이동 기록, 방문한 저장 장소 또는 근처 장소, 총 이동 시간/거리, 오래 머문 위치, 데이터 없음/권한 없음/오래된 데이터 상태를 표시한다.
- 거리/장소명 계산은 현재 프로젝트에 있는 데이터와 라이브러리만 사용한다. 외부 지도/역지오코딩 API를 새로 추가해야 하면 비용, 개인정보, 키 관리, 정책 리스크를 먼저 명시하고 승인 없이 임의 추가하지 않는다.
- 캘린더 월/일 UI에서 이동경로가 있는 날짜는 테마 색상 기반의 명확한 표시를 제공하고, 날짜 선택 후 요약 패널 또는 상세 화면에서 이동경로를 확인할 수 있게 한다.
- 이동경로 요약은 Playwright로 날짜 선택, 자녀 전환, 데이터 있음/없음 상태를 실제 화면에서 확인한다.

4. 주위소리 듣기:
- 선택된 아이에게만 주위소리 듣기 요청이 전달되는지 확인한다.
- Android 마이크 권한, notification 권한, foreground service microphone type, persistent notification, 원격 요청 상태, 실패/거부/오프라인 상태 처리를 확인한다.
- 아이 기기에서 주위소리 듣기에 필요한 권한과 설정을 사용자가 이해할 수 있게 안내하고, 권한 확보 플로우를 앱 안에서 완료할 수 있게 구현한다.
- 마이크 권한 거부, "다시 묻지 않음", 알림 권한 거부, 배터리 최적화 제한, 백그라운드 제한, 절전 모드, 네트워크 끊김, 기기 잠금, 화면 꺼짐, 무음 모드, 방해 금지 모드, 폴더블 기기 접힘 상태를 각각 진단한다.
- 무음 모드와 방해 금지 모드는 마이크 캡처 자체를 막는지, 재생/알림/권한 안내만 영향을 받는지 실제 기기에서 확인한다. OS 정책상 제한되는 경우 우회하지 말고 사용자가 바꿀 수 있는 설정 안내와 명확한 실패 상태를 제공한다.
- 폰이 접혀 있거나 화면이 꺼져 있거나 앱이 백그라운드에 있어도 Android 정책 안에서 foreground service와 persistent notification을 통해 세션이 유지되는지 확인한다.
- 주위소리 세션 시작, 권한 확인, 아이 기기 수락/준비, 스트림 연결, 부모 기기 재생, 종료, 실패 복구까지 단계별 상태를 UI에 표시한다.
- 부모 앱에는 "요청중", "아이 기기 권한 확인중", "연결중", "듣는중", "권한 필요", "기기 오프라인", "마이크 사용 불가", "OS 설정 확인 필요" 같은 상태를 구분해서 보여준다.
- 관련 아이 기기의 권한 확보 흐름이 제대로 동작하는지 실제 연결 기기에서 검증한다.
- 다른 아이에게 잘못 요청되거나 다른 아이 상태에 반영되는 문제를 차단한다.
- Google Play 정책, PIPA, 기존 AGENTS.md의 안전 요구사항을 지킨다. 몰래 녹음하거나 persistent notification 없이 마이크를 사용하거나 OS 개인정보 제한을 우회하는 구현은 금지한다.

5. 프로필/설정 저장 UX:
- 아이 이름 변경, 프로필 이미지/이모티콘 변경, 테마 색상 변경, 기타 설정 저장 흐름을 확인한다.
- 저장 버튼 클릭 시 저장중/로딩중/비활성화/성공/실패 상태가 명확히 표시되도록 수정한다.
- 중복 클릭으로 중복 저장되지 않게 한다.
- 저장 실패 시 사용자에게 복구 가능한 메시지를 보여준다.
- 저장 성공 후 DB, 로컬 상태, Realtime/UI 표시가 일관되게 맞는지 확인한다.
- 테마 색상은 저장 즉시 앱 전체에 반영되고 새로고침/앱 재실행 후에도 유지되어야 한다.

6. 아이콘, 테마 색상, 사용자 인지성:
- 회원가입 시 선택한 테마 색상이 앱 전체의 아이콘, 주요 버튼, 선택 상태, 탭, 칩, 캘린더 포인트, 상태 표시와 일관되게 연결되는지 확인한다.
- 아이콘이 지나치게 다양한 색을 쓰거나 기능별로 산만하게 보이면 테마 토큰 기반으로 정리한다.
- 아이콘만 보고도 기능 의미를 알 수 있는지 확인한다. 의미가 불명확한 곳은 접근 가능한 label, title, 보조 문구, 상태 문구를 추가한다.
- 버튼/아이콘/탭의 active, pressed, disabled, loading, error 상태가 명확히 구분되는지 확인한다.
- 모든 버튼과 아이콘 버튼은 클릭 시 텍스트 선택이 발생하지 않도록 user-select, 터치 영역, pointer/click 동작을 검증한다.
- 회원가입에서 테마 선택 후 실제 홈/오늘/캘린더/프로필 화면에 같은 테마가 반영되는지 Playwright로 확인한다.

7. 프로필 사진과 이미지 표시 성능:
- 앱에서 프로필 사진, 아이 이모티콘, 자녀 아바타가 느리게 표시되는 원인을 실제 코드 기준으로 찾는다.
- 이미지 URL 생성, Supabase storage 접근, 캐시 정책, state 초기화, 중복 fetch, 이미지 preload/lazy loading, fallback 표시, layout shift를 확인한다.
- 현재 프로젝트와 브라우저/Capacitor 환경에서 안전한 방식으로 이미지 표시를 최적화한다.
- 프로필 사진이 늦게 뜰 때 빈 화면으로 보이지 않도록 skeleton, placeholder, 기존 이모티콘 fallback, loading/error 상태를 제공한다.
- 이미지 로딩 실패, 권한 없음, storage object 없음, 네트워크 실패, child 전환 직후 stale image 표시를 방어적으로 처리한다.
- 최적화 후 Playwright 스크린샷과 실제 Android 기기에서 프로필 사진 표시 지연과 fallback 동작을 확인한다.

8. DB와 앱 기능별 최적화:
- 기능별 필요한 인덱스, RLS 정책, realtime publication, nullable/default, FK 관계를 확인한다.
- 위치 히스토리와 하루 이동경로 요약에 필요한 child_id/date/recorded_at 기준 인덱스, RLS, realtime publication을 확인한다.
- 필요한 DB 변경은 migration으로 작성하고 down migration을 포함한다.
- 불필요한 전체 fetch, 중복 Realtime 구독, 과도한 렌더링, child 전환 시 메모리 누수를 줄인다.
- 로딩, 빈 배열, null/undefined, 권한 없음, API 실패, Realtime 지연을 방어적으로 처리한다.
- 최적화는 기능 안정화 범위 안에서만 수행하고 대규모 리팩토링은 피한다.

- 연결된 Android 기기에 빌드/설치 후 권한, 위치, 주위소리 듣기, 저장 UX 확인
- 주위소리 듣기는 실제 아이 기기와 부모 기기 조합으로 다음 상태를 각각 확인한다: 권한 정상, 마이크 권한 없음, 알림 권한 없음, 기기 잠금, 화면 꺼짐, 앱 백그라운드, 무음 모드, 방해 금지 모드, 폴더블 접힘 상태, 네트워크 불안정, 배터리 최적화 제한.
- 각 상태에서 부모 앱 UI, 아이 기기 UI/알림, DB 상태, Realtime 이벤트, Android 로그를 확인하고, 안 들리는 경우 원인과 사용자 안내 문구가 정확한지 기록한다.

10. 다자녀 검증 시나리오:
- 테스트용 아이 A와 아이 B를 준비한다.
- 첫 화면에서 아이 A 선택:
  - 홈/오늘/위치/캘린더 하루 이동경로/프로필/테마/아이콘/프로필 사진/주위소리 듣기가 A 데이터만 사용하는지 확인한다.
- 첫 화면에서 아이 B 선택:
  - 같은 기능이 B 데이터만 사용하는지 확인한다.
- A에서 바꾼 위치, 하루 이동경로, 프로필, 테마, 저장 상태, 요청 상태가 B에 섞이지 않는지 확인한다.
- B에서 바꾼 데이터가 A에 섞이지 않는지 확인한다.
- DB 쿼리와 UI 모두에서 evidence를 남긴다.

11. 완료 기준:

- 주위소리 듣기 권한과 실행 흐름이 실제 Android 부모/아이 기기에서 검증된다.
- 주위소리 듣기는 아이 기기의 권한 부족, 무음/방해 금지, 화면 꺼짐, 백그라운드, 폴더블 접힘, 배터리 최적화, 네트워크 문제 등 상태별 실패 원인을 앱이 구분해서 안내한다.

- 프로필/설정 저장에는 저장중/성공/실패 UX가 있다.
- 테마/프로필 저장이 DB와 UI에 일관되게 반영된다.
- 회원가입/프로필에서 선택한 테마 색상이 아이콘과 주요 UI 상태에 일관되게 반영된다.
- 아이콘은 사용자가 의미를 이해하기 쉽고, 상태와 클릭 가능 여부가 명확하다.
- 프로필 사진과 자녀 아바타는 빠르게 표시되고, 느린 네트워크/실패 상태에서도 fallback과 loading UI가 안정적으로 동작한다.
- Playwright 스크린샷으로 실제 화면 표시가 확인된다.
- DB/RLS/Realtime 구성이 기능 요구사항을 실제로 커버한다.
- 테스트, 브라우저, Android 기기 검증 결과가 기록된다.
- 실패 로그, 미검증 항목, 불확실한 추정이 남아 있으면 완료하지 않는다.
```
