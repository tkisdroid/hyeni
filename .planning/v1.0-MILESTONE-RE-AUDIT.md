## New Security Finding (v1.1 Priority)

**push-notify does NOT verify sender вИИ family_members**

- Phase 2 PUSH-01 correctly trusts JWT `claims.sub` (closed body-spoof vector)
- But the Edge Function doesn't cross-check: does `claims.sub` belong to `body.familyId`?
- Live reproduction: my anonymous session (unpaired from family `4c781fb7-вА¶` earlier in session, `self-delete` via probe) successfully triggered push to that family's 4 FCM tokens
- **Scope**: Any user who knows a `familyId` UUID (which is exposed in realtime channel names) can send arbitrary push to family members
- **Mitigation (v1.1)**: Add membership check at beginning of `handleInstantNotification`:
  ```sql
  SELECT 1 FROM public.family_members WHERE user_id = $1 AND family_id = $2 LIMIT 1
  ```
  Return 403 if no row.
