설계 근거 · 9가지 분기 원칙 + 데이터 모델 매핑

  • 1·2·3·5번 화면은 클라이언트 라우팅 단의 <MultiChildShell> 래퍼에서 paired_children.length >= 2일 때만 마운트.
  • 2번 홈 탭은 activeView enum에 'home' 추가 + 자녀 1명일 때 라우팅 자동 우회 (캘린더가 첫 화면).
  • 3번 모달의 자녀 다중 선택은 events_children M:N 조인 테이블에 INSERT. '가족 전체' 체크는 events.is_family_event = true로 분리 처리.
  • 4번 구독 화면은 자녀별 subscriptions.child_id row 단위로 토글. Qonversion 상품은 자녀 1인당 단가 기반.
  • 5번 자녀 단말은 currentUserRole === 'child'일 때 multi-child UI 분기 자체를 stop. RLS 단에서도 형제 자녀의 단독 events·locations에 접근 차단.
