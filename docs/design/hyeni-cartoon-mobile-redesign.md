# 혜니캘린더 카툰 모바일 리디자인 저장본

> 목적: 현재 구현된 혜니캘린더 앱의 화면 구성을 유지하면서, 출시 전 검토에서 확정한 **카툰형 모바일 친화 디자인**을 코드 적용 가능한 기준으로 저장한다.

## 1. 디자인 방향

- 100% 모바일 사용을 전제로 한다.
- 데스크톱형 다단 레이아웃을 금지하고, 모든 화면을 1열 세로 스크롤 중심으로 구성한다.
- 기존 앱 스크린샷의 정보 구조는 유지한다.
- 이전 작업에서 선호도가 높았던 카툰 디자인을 기준으로 한다.
- 브랜드 정서: 아이 친화적, 가족 안전, 부드러움, 신뢰감.
- 메인 색상: 크림 배경 + 핑크 강조 + 흰색 카드.
- 주요 UI 성격: 둥근 카드, 큰 터치 영역, 부드러운 그림자, 카툰 일러스트, 명확한 하단 내비게이션.

## 2. 확정된 핵심 규칙

### 모바일 레이아웃

- 기준 뷰포트: 390 × 844.
- 최소 대응 폭: 360px.
- 주요 버튼 높이: 최소 48px, 권장 54~60px.
- 입력 필드 글자 크기: 16px 이상. iOS 자동 줌 방지.
- 하단 내비게이션 고정 사용 시 페이지 하단 padding을 반드시 확보한다.
- 지도, 달력, 일정 카드는 1열 카드로 배치한다.
- 긴 주소, 일정명, 자녀명은 `ellipsis` 처리한다.

### 페어링 코드

- 최종 형식: `KID-E5O3KW3`.
- UI에서는 `KID-` prefix를 고정 영역으로 분리하고, 뒤 코드는 대문자·숫자 입력 영역으로 둔다.
- 입력 예시는 `E5O3KW3`.

### 로그인 버튼

학부모 로그인에는 다음 버튼을 포함한다.

1. 카카오로 시작
2. 구글로 시작
3. 네이버로 시작
4. ID/PW로 로그인

## 3. 화면별 리디자인 저장 목록

### 인증 / 가입

- 스플래시 + 역할 선택
  - 학부모 / 아이 카드 2개.
  - 마지막 선택 역할 배너 표시.
- 학부모 로그인
  - 카카오, 구글, 네이버 로그인.
  - ID/PW 로그인 확장 카드.
- 학부모 가입
  - 이름, 아이디, 비밀번호, 비밀번호 확인, 역할, 생년월일, 전화번호, 인증번호 버튼.
  - 생년월일은 모바일 바텀시트 picker.
- 아이 가족 합류
  - `KID-` prefix 분리형 입력.
  - QR 연결 버튼.
- 권한 설정
  - 위치 항상 허용, 알림 허용, 배터리 예외, 정확한 알림.
  - 필수/권장 pill 표시.

### 학부모

- 부모 홈
  - 푸시 알림 안내 배너.
  - 혜니캘린더 요약 카드.
  - 자녀 관리 카드.
  - 현재 위치 지도 카드.
  - 오늘 일정 shortcut.
- 자녀 상세 / 우리아이
  - 큰 지도, 위치 trail, 위치 새로고침.
  - 배터리, 마지막 업데이트, 화면 켜짐, 응답 상태.
  - 오늘 일정 timeline.
- 일정 화면
  - 월간/주간 캘린더.
  - 날짜별 timeline.
  - 일정 카드에서 길찾기.
- 장소관리
  - 학원 / 자주 가는 장소 / 위험지역 탭.
  - 학원 카드, 저장 장소, 위험지역 반경 slider.
- 가족 메모
  - 가족 채팅형 메모.
  - 빠른 답장 chip.
  - 읽음, 시간, 실시간 동기화 상태.
- 칭찬 스티커북
  - 일찍 / 정시 / 아쉬워 / 칭찬 집계.
  - 월별 sticker grid.
  - 부모 스티커 보내기.
- 부모 설정
  - 내 계정, 자녀 관리, 알림, 구독, 데이터·개인정보, 도움말, 위험 영역.
- 프리미엄 paywall
  - 7일 무료 체험.
  - 월 1,500원 / 연 15,000원.
  - 4개월 무료 표시.
  - Google Play 자동 갱신 안내.

### 아이

- 아이 홈
  - 현재 시간 label.
  - 동적 hero copy.
  - 다음 일정 카드 + 길찾기.
  - 현재 위치 카드.
  - 메모, 스티커, 전화, 친구놀이 2×2 quick action.
  - 집으로 가기 / 다음 일정으로 가기.
- 아이 일정
  - 자녀 친화 timeline.
  - 지금! ribbon.
  - 도착 완료 / 조금 늦어요 badge.
  - 길찾기 bottom sheet.
- 자녀 설정
  - 테마 6색 picker.
  - 동물 캐릭터 8종.
  - 소리, 진동, 마스코트 토글.
  - 부모님께 변경 요청.

## 4. 적용 우선순위

1. 인증/온보딩: RoleSelect, ParentAuth, ParentSignup, ChildPairInput, PermissionSetup.
2. 핵심 사용: ParentHome, ChildDetail, Calendar, ChildHome.
3. 관리 기능: PlaceManager, FamilyMemo, Stickerbook.
4. 설정/수익화: ChildSettings, ParentSettings, Paywall.

## 5. 컴포넌트 전환 원칙

기존 Supabase, 권한, 위치, 일정, 메모 로직은 유지한다. JSX 구조와 스타일 계층만 교체한다.

공통 컴포넌트 후보:

- `HyPage`
- `HyCard`
- `HyButton`
- `HyBottomNav`
- `HyNoticeBanner`
- `HyParentHeader`
- `HyChildChipRail`
- `HyMetricCard`
- `HyScheduleTimelineItem`
- `HyBottomSheet`

## 6. 검수 기준

- 360px 폭에서 좌우 잘림 없음.
- 390×844 기준 주요 CTA가 엄지 영역에 위치.
- 카툰 이미지는 실제 텍스트를 포함하지 않는다.
- 실시간 데이터 영역은 skeleton/loading 상태가 있어야 한다.
- 바텀시트는 `max-height: 85dvh` 이하.
- Android WebView와 iOS Safari에서 입력 필드 확대 발생 없음.
- Playwright 캡처 기준: 390×844, 430×932.
